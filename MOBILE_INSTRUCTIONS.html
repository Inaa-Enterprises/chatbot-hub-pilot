Follow these instructions to deploy your Synapse backend and access it on any mobile device through the internet.

Important: This guide assumes you have basic familiarity with GitHub and web deployment. If you're new to this, take your time and follow each step carefully.

üìã Required Files

Your project should include these essential files:

  • main.py - Main Flask application
  • requirements.txt - Python dependencies
  • Procfile - Application startup command
  • personas.py - AI persona definitions
  • agent_routes.py - API routes for agents
  • enhanced_mobile_client.html - Mobile client interface
  • run.py - Script to run the application locally

These files are all already in your project directory.

üöÄ Deployment Steps

1

Create a GitHub Repository

If you haven't already, create a new repository on GitHub:

  1. Go to github.com/new
  2. Name it something like "synapse-backend"
  3. Make it public (private repos have limits on free services)
  4. Don't initialize with a README
2

Push Your Code to GitHub

Open a terminal in your project directory and run:

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/synapse-backend.git
git push -u origin main

Replace YOUR_USERNAME with your actual GitHub username.

3

Deploy to Render

Render is a free service that's easy to set up:

  1. Go to render.com and sign up with GitHub
  2. Click "New +" and select "Web Service"
  3. Connect your GitHub repository
  4. Fill in these settings:
    • Name: synapse-backend
    • Region: Choose closest to you
    • Branch: main
    • Runtime: Python 3
    • Build Command: pip install -r requirements.txt
    • Start Command: gunicorn main:app
  5. Click "Create Web Service"
4

Wait for Deployment

Deployment will take 5-10 minutes. You'll see logs in real-time. When complete, you'll get a URL like:

https://synapse-backend-xxxx.onrender.com

This is your public backend URL.

üì± Mobile Access

1

Get the Mobile Client

You need the mobile client HTML file on your phone:

Tip: The easiest way is to email the file to yourself.
  1. Email enhanced_mobile_client.html to your email
  2. Open the email on your phone
  3. Download the attachment
  4. Open it with any web browser
2

Connect to Your Backend

  1. Open the mobile client HTML file in your phone's browser
  2. Enter your Render URL in the "Backend URL" field
  3. Click "Connect to Backend"
  4. You should see a success message
3

Start Chatting

You can now:

  • Switch between different AI personas
  • Send messages and get responses
  • Use the app in both portrait and landscape modes
  • Works on both regular phones and foldable devices

‚ö° Quick Testing Option

If you want to test without deploying:

  1. Run your app locally: 
    python run.py --host 0.0.0.0 --port 8000
  2. In another terminal, expose it with ngrok: 
    # Install ngrok first
npm install -g ngrok

# Expose your local server
ngrok http 8000
  3. Use the ngrok URL in your mobile client
Note: For local testing, your phone and computer must be on the same network.

‚ùì Troubleshooting

Common Issues:

  • "Application Error" on Render: Check logs in Render dashboard
  • Can't connect from mobile: Verify URL uses https:// and is accessible
  • Personas not loading: Check that backend is running and accessible
  • Slow first load: Normal for free tier services that sleep after inactivity
Pro Tip: Bookmark your mobile client page for easy access later.
